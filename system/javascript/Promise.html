# Promise

# 1. 概述

Promise 是异步编程的一种解决方案，可以替代传统的解决方案—回调函数和事件。ES6 统一了用法，并原生提供了 Promise 对象。

作为对象，Promise 有以下两个特点：

  • 对象的状态不受外界影响。

  • 一旦状态改变了就不会在变，也就是说，任何时候 Promise 都只有一种状态。


# 2. Promise 的状态

Promise 有三种状态，分别是：

  • Pending：进行中。

  • Resolved：已完成。

  • Rejected：已失败。

Promise 从 Pending 状态开始，如果成功就转到成功状态，并执行 resolve 回调函数；如果失败就转到失败状态并执行 reject 回调函数。


# 3. 基本用法

可以通过 Promise 的构造函数创建 Promise 对象：

const promise = new Promise((resolve, reject) => {
  // ...
})

Promise 构造函数接收一个函数作为参数，该函数的两个参数是 resolve 和 reject，它们由 JavaScript 引擎提供。

  • resolve：Promise 成功后的回调函数，并将结果作为其参数传递出去。

  • reject：Promise 失败后的回调函数，并将错误作为其参数传递出去。

function greet () {
  const promise = new Promise((resolve, reject) => {
    const greet = 'hello  world!'
    resolve(greet)
  })
  return promise
}

greet().then(res => {
  console.log(res)
})

注意

创建一个 Promise 对象时会立即执行里面的代码，为了更好的控制代码的运行时机，可以将其包含在一个函数中，并将这个 Promise 作为函数的返回值。


# 4. Promise 的 then 方法

Promise 的 then() 有三个参数：成功回调，失败回调和前进回调。一般情况下只需要实现第一个，后面是可选的。

Promise 中最为重要的是状态，通过 then() 的状态传递可以实现回调函数链式操作。

先执行以下代码：

function greet () {
  const promise = new Promise((resolve, reject) => {
    const greet = 'hello  world!'
    resolve(greet)
  })
  return promise
}

const p = greet().then(res => {
  console.log(res)
})

console.log(p)

输出结果为：

从中可以看出 promise 执行 then() 后，返回的是另一个 Promise 对象，并且 Promise 的执行是异步的，因为 hello world! 是在最后打印出来的。

因为 promise 执行 then() 后返回的是另一个 Promise 对象，所以就可以根据这一特性，不断的链式调用回调函数：

function greet () {
  const promise = new Promise((resolve, reject) => {
    const greet = 'hello  world!'
    resolve(greet)
  })
  return promise
}

greet().then(res => {
  console.log(res + 1)
  return res
}).then(res2 => {
  console.log(res2 + 2)
  return res2
}).then(res3 => {
  console.log(res3 + 3)
})

注意

  • then() 在链式调用中，回调函数会把上一个 then() 中返回的值当做参数值供当前 then() 调用。

  • then() 执行完毕后需要返回一个新的值给下一个 then() 调用，没有返回值则默认返回 undefined。

  • 每个 then() 只能使用前一个 then() 的返回值。


# 5. Promise 的 catch 方法

Promise 的状态变为 reject 时，会被 catch() 捕捉到。

注意

如果前面设置了 reject 方法的回调函数，则 catch() 不会捕捉到状态变为 reject 的情况。

如果在 resolve 或者 reject 时发生错误，也会被 catch() 捕捉到，这与 java，c++ 的错误处理方式相同，这样可以避免程序卡死在回调函数中。

一般情况下，不要在 then() 中定义 Reject 的回调函数，尽量使用 catch() 来处理 Reject 错误。

function judgeNumber (num) {
  const promise = new Promise((resolve, reject) => {
    num = 5
    if (num < 5) {
      resolve('num小于5，值为：' + num)
    } else {
      reject(new Error('num不小于5，值为：' + num))
    }
  })
  return promise
}

judgeNumber().then(res => {
  console.log(res)
}).catch(err => {
  console.log(err)
})

# 6. Promise 的其他方法

# finally()

finally() 用于指定不管 Promise 对象最后状态如何，都会执行的操作。

promise.then(res => {
// ...
}).catch(err => {
// ...
}).finally(() => { 
// ...
})

上面代码中，不管 promise 最后的状态，在执行完 then() 或 catch() 指定的回调函数以后，都会执行 finally() 指定的回调函数。

# all()

Promise 的 all() 提供了并行执行异步操作的能力，是在所有异步操作执行完后才执行回调。可以将多个 Promise 实例，包装成一个新的 Promise 实例。

const p = Promise.all([p1, p2])

上面代码中，Promise.all() 接受一个数组作为参数，p1、p2 都是 Promise 实例。

p 的状态由 p1、p2 决定，分成两种情况：

  • 只有 p1、p2 的状态都变成 fulfilled，p 的状态才会变成 fulfilled，此时 p1、p2 的返回值组成一个数组，传递给 p 的回调函数。

  • 只要 p1、p2 之中有一个 Rejected，p 的状态就变成 Rejected，此时第一个 Rejected 的实例返回值，会传递给 p 的回调函数。

注意

如果作为参数的 Promise 实例，自己定义了 catch()，那么它一旦被 Rejected，并不会触发 Promise.all() 的 catch()。

const p1 = new Promise((resolve, reject) => {
  resolve('hello')
}).then(res => res).catch(err => err)

const p2 = new Promise((resolve, reject) => {
  throw new Error('报错了')
}).then(res => res).catch(err => err)

Promise.all([p1, p2]).then(res => console.log('成功：' + res)).catch(err => console.log('失败' + err))

// 打印结果：成功：hello,Error: 报错了

上面代码中，p1 会 Resolved，p2 首先会 Rejected，但是 p2 有自己的 catch()，该方法返回的是一个新的 Promise 实例，p2 实际上指向的是这个实例。

该实例执行完 catch() 后，也会变成 Resolved，导致 Promise.all() 参数里面的两个实例都会 Resolved，因此会调用 then() 指定的回调函数，而不会调用 catch() 指定的回调函数。

# race()

all() 是等到所有的 Promise 都执行完，再来执行回调函数。race() 则不同，只有一个实例率先改变状态，就开始执行回调函数。

const p = Promise.race([p1, p2])

上面代码中，只要 p1、p2 中有一个实例率先改变状态，p 的状态就跟着改变。那个率先改变的 Promise 实例的返回值，就传递给 p 的回调函数。

# allSettled()

当不关心异步操作的结果，只关心这些操作有没有结束时，可以使用 Promise.allSettled() 。

如果没有这个方法，想要确保所有操作都结束，就很麻烦。Promise.all() 无法做到这一点。

// 示例：等到三个请求都结束，不管请求成功还是失败，loading 图标都会消失。
const promises = [
  fetch('/api-1'),
  fetch('/api-2'),
  fetch('/api-3'),
];

await Promise.allSettled(promises)
removeLoadingIndicator()

# any()

ES2021 引入了 Promise.any()。该方法接受一组 Promise 实例作为参数，包装成一个新的 Promise 实例返回。

只要参数实例有一个变成 fulfilled 状态，包装实例就会变成 fulfilled 状态；如果所有参数实例都变成 Rejected 状态，包装实例就会变成 Rejected 状态。

const promises = [
  fetch('/endpoint-a').then(() => 'a'),
  fetch('/endpoint-b').then(() => 'b'),
  fetch('/endpoint-c').then(() => 'c'),
]

try {
  const first = await Promise.any(promises);
  console.log(first)
} catch (error) {
  console.log(error)
}
