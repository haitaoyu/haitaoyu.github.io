# async & await

# 1. 简介

async 函数是 ES8 引入的新特性，主要目的是为了简化使用 Promise 对象时的语法。

使用 async 和 await 可以更简洁的写出基于 Promise 对象的异步行为，而无需刻意地链式调用 Promise 对象。可以用同步的写法写异步的代码。


# 2. async

async 函数返回的是一个 Promise 对象，可以使用 then 方法添加回调函数。

只要使用 async，不管函数内部返回的是什么类型的数据，都会被包装成 Promise 对象。

async function helloAsync () {
  return 'helloAsync'
}

console.log(helloAsync()) // Promise {<fulfilled>: 'helloAsync'}

helloAsync().then(res => {
  console.log(res) // helloAsync
})

# 3. await

await 用于等待右侧表达式的结果，一般是等待一个 Promise 对象的结果。如果等待的不是 Promise 对象，则返回该表达式的结果。

await 有以下几个特性：

  • await 想要在函数中使用，只有在 async 函数中使用才有效，在普通函数中使用会报错。


  • await 在等待右侧表达式结果的时候，会阻塞后面的代码。

    示例 1：

    function test () {
  return new Promise(resolve => {
    setTimeout(() => resolve('test'), 2000)
  })
}

async function test2 () {
  const result = await test()
  console.log(result)
  console.log('test2') 
}

test2()

// 打印结果：
// 由于 test2() 中 await 造成的阻塞，两秒后 test2 在 test 之后被打印。

    示例 2：

    function test () {
  return new Promise(resolve => {
    setTimeout(() => resolve("test"), 2000)
  });
}

const result = await test()

console.log(result)
console.log('end')

// 打印结果：
// 由于 test() 造成的阻塞，end 两秒后才会被打印

  • async 函数中的 await 只会阻塞 async 函数中的代码，不会阻塞整体代码执行。

function test () {
  return new Promise(resolve => {
    setTimeout(() => resolve('test'), 2000)
  })
}

async function test2 () {
  const result = await test()
  console.log(result)
}

test2()
console.log('end')

// 打印结果：
// end 先被打印，两秒后 test 被打印

# 4. 执行顺序

运行 async 函数时，遇到 await 会阻塞 async 函数后面的代码，然后去执行 async 外面的同步代码。同步代码执行完，再回到 async 内部，继续执行 await后面的代码。

以下面的代码分析：

async function test1 () {
  console.log('start test1')
  console.log(await test2())
  console.log('end test1')
}

async function test2 () {
  console.log('test2')
  return await 'return test2 value'
}

test1()

console.log('start async')

setTimeout(() => {
  console.log('setTimeout')
}, 0)

new Promise((resolve, reject) => {
  console.log('promise1')
  resolve()
}).then(() => {
  console.log('promise2')
})

console.log('end async')

执行结果：

  • 首先执行宏任务，执行 test1()，执行 console.log('statr test1')。

  • 遇到 await，先执行右边 test2() 中的 console.log('test2')，中断了后面的代码，执行 test1() 外面的同步代码。

  • 执行 console.log('start async')。

  • 遇到 setTimeout，推到到下个宏任务队列中 。

  • 执行 Promise 里面的同步代码 console.log('promise1')。

  • 运行到 promise().then，发现是 Promise 对象，推到微任务队列中。

  • 执行 console.log('end async')。

  • test1() 外面的同步代码执行结束后，回到 test1() 中，console.log(await test2()) 执行完成后返回 Promise {<resolved>: "return test2 value"}，是Promise 对象，推到微任务队列中。

  • 此时第一个宏任务结束，开始执行所有微任务，因为微任务队列先进先出，所以先执行 console.log('promise2')，后执行 console.log('return test2 value')。

  • test2() 执行完后，后面的代码不再阻塞，执行 console.log('end test1')。

  • 执行下个宏任务，即执行 console.log('setTimeout')。

补充

宏任务和微任务都是队列，宏任务有 script、setTimeout、setInterval 等，微任务有 Promise.then、Promise.catch、Promise.finally、process.nextTick 等。

宏任务和微任务的关系如下：

先执行第一个宏任务，执行结束后，执行所有的微任务，然后再执行下个宏任务。


# 5. 捕获异常

# catch

async function fn () {
  await new Promise((resolve, reject) => {
    setTimeout(() => {
      reject('err')
    }, 1000)
  })
}

fn().catch(alert)

# try/catch

async function fn () {
  try {
    await new Promise((resolve, reject) => {
      setTimeout(() => {
        reject('err')
      }, 1000)
    })
  } catch (err) {
    alert(err)
  }
}

fn()

# 6. 优缺点

# 优点

相对于 Promise 对象，async/await 处理 then 的调用链，代码要清晰很多，几乎和同步代码一样。

# 缺点

滥用 await 可能会导致性能问题，因为 await 会阻塞代码。
