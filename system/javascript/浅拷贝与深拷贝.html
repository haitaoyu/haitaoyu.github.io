# 浅拷贝与深拷贝

只有引用数据类型才有浅拷贝和深拷贝之分，基本类型数据是没有浅拷贝和深拷贝之分的。

# 1. 浅拷贝

浅拷贝只复制指向某个对象的指针，不复制对象本身，新对象和原对象还是共享同一块内存，修改新对象会影响原对象。

直接使用 = 赋值就可以实现浅拷贝。


# 2. 深拷贝

深拷贝会创建一个一模一样的对象，新对象和原对象不共享内存，修改新对象不会影响原对象。

实现深拷贝的几种方式：

  • JSON.parse(JSON.stringify())

    const arr = [1, 3, { username: 'hayden' }]
const arr2 = JSON.parse(JSON.stringify(arr))
arr2[2].username = 'haitao'

console.log(arr === arr2) // false

    先使用 JSON.stringify() 将对象转成 JSON 字符串，再使用 JSON.parse() 把字符串解析成对象，一去一来，新的对象就产生了，而且新的对象会开辟新的栈，实现深拷贝。

    这种方法虽然可以实现数组或对象的深拷贝，但是不能处理数组或对象中的函数和正则，因为这两者被 JSON.stringify() 和 JSON.parse() 处理后，函数会变为 null，正则会变为空对象。如下例：

    const arr = [1, 3, { username: 'hayden' }, function () { }]
const arr2 = JSON.parse(JSON.stringify(arr))
arr2[2].username = 'haitao'

console.log(arr, arr2)


  • 使用函数库 lodash 的 cloneDeep() 方法。

    lodash 提供了 cloneDeep() 来实现引用类型的深拷贝：

    import { cloneDeep } from 'lodash-es'

const obj1 = { a: 1, b: { f: { g: 1 } }, c: [1, 2, 3] }
const obj2 = cloneDeep(obj1)

console.log(obj1.b.f === obj2.b.f) // false

  • 使用拓展运算符 + 解构赋值。

    const obj = { name: 'haitao', age: 18 }
const obj2 = { ...obj }

console.log(obj === obj2) // false

    拓展运算符 + 解构赋值有一定的局限性：

    • 如果数组或对象中的元素都是基本类型，那么可以进行深拷贝，如上例。

    • 如果数组或对象中的元素包含引用类型，那么对引用类型的拷贝是浅拷贝。

      const obj = { name: 'haitao', age: 18, address: { city: 'Shanghai' } }
const obj2 = { ...obj }
obj.address.city = 'Beijing'

console.log(obj2.address.city) // Beijing

  • 手写递归方法。

    递归方法实现深度克隆原理：遍历对象或数组，直到最后一层都是基本数据类型，然后再去复制。

    function deepClone (obj) {
  const objClone = Array.isArray(obj) ? [] : {}
  if (obj && typeof obj === 'object') {
    for (const key in obj) {
      if (obj.hasOwnProperty(key)) {
        // 判断ojb子元素是否为对象，如果是，递归复制
        if (obj[key] && typeof obj[key] === 'object') {
          objClone[key] = deepClone(obj[key])
        } else {
          // 如果不是，简单复制
          objClone[key] = obj[key]
        }
      }
    }
  }
  return objClone
}
const a = [1, 2, 3, 4]
const b = deepClone(a)
a[0] = 2
console.log(a, b) // [2, 2, 3, 4] [1, 2, 3, 4]
