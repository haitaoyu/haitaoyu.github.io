# Git 的使用

# 1. 介绍

Git 是一款分布式源代码管理工具，在本地和远程服务器上都有 Git 版本库。

要想熟练使用 Git，必须理解 Git 的工作原理：

本地仓库中，工作区文件需要先被添加进暂缓区，然后暂缓区提交到本地分支，最后提交到远程分支。

# 2. 使用流程

  1. 在远程版本库上创建一个共享版本库。

  2. 开发人员使用 git clone 命令下载远程版本库。

  3. 进入开发阶段。

    1）在 Git config 文件中添加使用者信息（必须设置，否则无法提交代码）：

    git config user.name '姓名'

git config user.email '邮箱'

    2）添加文件到暂缓区：

    git add .

    3）将编写好的代码提交到本地分支：

    git commit -m'说明'

    4）将代码提交到远程分支：

    git push origin 分支名称

    5）获取远程分支的最新内容：

    git pull

  4. 注意事项：

    • 不要将不能运行的代码提交到远程分支。

    • 提交代码到远程分支时，要先 git pull 最新代码，再 git push 我们自己的代码。

    • 如果 git pull 下来的代码有冲突，需要手动解决冲突后再 git push 代码。


# 3. 分支

  • 查看版本库分支：

    -- 查看本地版本库分支
git branch

-- 查看远程版本库分支
git branch -r

  • 创建分支：

    分支之间是独立的，不会相互影响。

    git branch 分支名称

  • 切换分支：

    切换到哪个分支，代码就会提交到哪个分支。

    git checkout 分支名称

  • 分支提交到远程版本库：

    git push origin 分支名称

  • 合并分支：

    例：在 master 分支中执行 git merge staging，代表是将 staging 中的代码都合并到 master 中。

    git merge 分支名称

  • 删除分支：

    -- 删除本地分支
git branch -d分支名称

-- 删除远程分支
git push origin --delete分支名称

  • 拉取指定分支的代码：

    git clone -b 分支名称 git地址

# 4. 撤销和回退

  • 撤销本地 commit：

    git reset --soft HEAD^

  • 远程分支回退到某个版本：

    -- 查看提交历史记录
git log


-- 本地分支回滚到指定版本
git reset --hard <commit ID号>


-- 强制推送到远程分支
git push origin HEAD --force

# 5. 忽略提交

如果不想让某些文件被 git，可以使用 touch .gitignore 命令创建 .gitignore 文件，在里面添加相应的忽略规则即可。

常用的忽略规则
.DS_Store
node_modules
/dist


# local env files
.env.local
.env.*.local

# Log files
npm-debug.log*
yarn-debug.log*
yarn-error.log*
pnpm-debug.log*

# Editor directories and files
.idea
.vscode
*.suo
*.ntvs*
*.njsproj
*.sln
*.sw?
