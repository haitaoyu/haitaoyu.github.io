# 二、部署到 GitHub Page

# 1. 创建 GitHub 仓库

登录 GitHub，点击 new repository，然后在 Repository name 填写 你的用户名 + .github.io 。

注意

仓库名必须为 你的用户名 + .github.io 。

# 2. 配置本地推送到远程仓库的权限

进入 git-bash 界面，然后做以下操作：

  1. 输入 git config --global --list ，验证邮箱是否与 GitHub 创建仓库的账号邮箱一致。 若不一致的话，通过下面命令进行修改。

    git config --global user.name "你的用户名"        // 设置全局用户名
git config --global user.email "你的登陆邮箱"     // 设置邮箱

  2. 在本地生成 ssh 秘钥，并拷贝秘钥。

    cd ~/.ssh
ssh-keygen -t rsa -C "你的email" // 生成秘钥
clip < ~/.ssh/id_rsa.pub // 复制密钥到剪贴板

  3. 打开 GitHub 页面，点击右上角的头像，点击下拉菜单的 setting，跳转页面后点击侧边栏的 SSH and GPG keys ，填入秘钥。

  4. 测试是否可以连接上 GitHub 。

    ssh -T git@github.com

# 3. 编写打包上传脚本

  1. 在 deploy.sh 文件中添加以下代码：

    # 确保脚本抛出遇到的错误
set -e

# 打包生成静态文件
npm run build

# 进入打包好的文件夹
cd docs/.vuepress/dist

# 创建git的本地仓库，提交修改
git init
git add -A
git commit -m 'deploy'

# 覆盖式地将本地仓库发布至github，因为发布不需要保留历史记录
# 格式为：git push -f git@github.com:'用户名'/'仓库名'.git master
git push -f git@github.com:haitaoyu/haitaoyu.github.io.git master

cd -

  2. 修改 package.json，在里面添加一条执行脚本文件的命令：

    "scripts": {
    ......
    "deploy": "bash deploy.sh"
}

之后每次部署的时候，只需要在 git-bash 界面运行 npm run deploy 即可实现打包上传操作。

GitHub 会自动更新页面代理，等待一会后再打开https://用户名.github.io，就可以看到页面更新了。
